快应用简介 & 快速上手指南

2019.03.07 hysunny quickapp  热度 ℃

快应用简介

快应用是基于手机硬件平台的新型应用形态，由主流手机厂商组成的快应用联盟共同制定标准并实现。使用前端技术栈开发，原生渲染，同时具备HTML5页面和原生应用的双重优点，其无需安装，即点即用。

快应用优势

  1. 流量
    目前有十大厂商支持：华为、小米、oppo、vivo、中兴、金立、联想、魅族、努比亚,一加，覆盖超过10亿设备；
  2. 无需安装，即点即用
    快应用初始大小限定为1M，采用分包下载，优先下载需要的部分，同时使用应用缓存，离线可用/异步更新；
  3. 多入口
    在系统搜索，智能助手，智能推荐，应用市场，浏览器、短信等位置都有入口，易于推广。
  4. 原生UI渲染，可享受原生应用的体验
  5. 提供桌面icon和push服务，留存高
  6. 提供更多底层可供调用的接口
    包括但不限于推送，统计、发送短信，可实现更多纯h5无法实现的功能
  7. 反垄断
    小程序腾讯有绝对控制权，对开发者有诸多限制；
  8. 与操作系统深度集成，能探索新型应用场景

快应用劣势

  1. 对包有大小限制，目前为1M，暂不支持复杂应用;
  2. 仅联合了10大厂商，其他如iphone，无法使用，损失了一波用户;
  3. 各大厂商联盟对快应用的支持程度也不一样，目前仅仅只有小米、华为、OPPO、VIVO入口支持较好;
  4. 社交：快应用缺乏微信的社交场景能力和传播手段，推广拉新成本仍不低。

快应用开发上线流程

Tips:

  1. 更新机制：快应用上架后，每次版本更新用户无感知
    新版本上架后，一般退出快应用后再次打开，应该就可以更新到新版本。
    当前快应用更新策略为：首次进入只是检测升级，下次再进入升级到版本；升级之后，桌面图标不会自动更新，即已经有的桌面图标不会更新，需删除图标之后，重新添加桌面之后打开。但是对于新用户，则会创建新的桌面图标。
  2. 小米快应用上线注意事项
  3. 华为快应用上架问题
  4. 各大厂商快应用数据查询汇总

7. 调试

打开快应用调试器界面如下图：

调试方法参考： https://doc.quickapp.cn/tutorial/overview/debug.html

注：
(1) 可以进行运行平台的切换，已小米为例，默认选择「快应用预览版」，切换到「快应用服务框架」会以线上平台运行本地快应用，如预览版不能测试的PUSH等功能可以切换到服务框架来测试。
(2) 将rpk包上传至CDN，不用每次都下载后再本地安装，可以在下载地址的URL好后拼接 ?raw=true， 然后使用二维码生成器生成二维码，再用调试器的扫码安装就可以。（当然，这里写个脚本或实现个工具更方便）。

8. 生成证书

快应用IDE -> Hap -> 生成证书 -> 填写签名信息 -> 工程的 sign 目录下会自动生成 release 目录，里面包含两个文件：private.pem 和 certificate.pem

9. 发布程序包

会在项目的 dist 目录下生成一个 ${package}.release.rpk 文件，这就是最后的包产物。
